# Solution
🎯 **問題分析：離線時的衝突防護**

當 Clipper 前端與 Signal Server 中斷連線時，目前雖然多數資料修改操作會檢查 WebSocket 狀態並阻擋，但仍有潛在風險：
- 某些 P2P DataChannel 的路徑可能繞過 WS 檢查  
- 中斷期間若仍然允許局部操作，重新連線後會與伺服器狀態產生衝突  
- UI 未明確提示用戶「目前處於唯讀模式」，使用者可能誤觸  

你的需求是：**斷線後所有協作功能強制唯讀、明確提示、防止任何可能導致衝突的資料變更**。

這是一個明確定義的改動，我可以直接拆成可執行的卡片。

## 📋 實施計畫（Task Cards）

### Card 1: 建立 `readOnly` 狀態與全域唯讀開關
| 項目 | 內容 |
|------|------|
| **目標** | 在 `APP.state` 加入 `readOnly` 旗標，並實作函式 `setReadOnly(bool)`，負責啟用/關閉唯讀介面。 |
| **實作範圍** | `clipper.html` 中的 `APP` 物件與 UI 相關程式碼。 |
| **驗收條件** | - `APP.state.readOnly` 初始值為 `false` <br> - 提供 `setReadOnly(true)` 時，所有會觸發資料變更的 UI 元件必須 `disabled`（按鈕、輸入框、檔案拖放區等） <br> - 顯示一個全頁頂端橫幅「🛑 信令伺服器中斷 — 唯讀模式」，類似 `errorToast` 但持續存在 <br> - `setReadOnly(false)` 時恢復所有元件可用、移除橫幅 <br> - 若 `setReadOnly(true)` 時有開啟的 modal 表單（如新增公告），應強制關閉 |
| **相依** | 無 |

### Card 2: 將連線狀態變更與唯讀模式綁定
| 項目 | 內容 |
|------|------|
| **目標** | 在 WebSocket 的 `onclose`、`onopen`、`onerror` 以及 `handleWsMessage` 中適當呼叫 `setReadOnly`，使斷線自動進入唯讀，重連成功後自動解除。 |
| **實作範圍** | `connectWebSocket` 內的 `ws.onclose`、`ws.onopen`、`ws.onerror`，以及 `handleWsMessage` 中 `'joined'` case 與 `autoReconnect` 成功後。 |
| **驗收條件** | - WS 斷線時（`onclose` 觸發，且非手動中斷） → 進入唯讀模式 <br> - 初始頁面尚未連線時（`updateStatus('disconnected')`） **不** 進入唯讀模式（因為根本無房間） <br> - 重連成功（`joined` 訊息收到）後 → 退出唯讀模式 <br> - 手動點擊「中斷」按鈕（`disconnect()`）後 → 退出唯讀模式（因為使用者主動離開，不需要唯讀保護） |
| **相依** | Card 1 |

### Card 3: 修改各功能事件，加入唯讀攔截
| 項目 | 內容 |
|------|------|
| **目標** | 在現有功能觸發處（點擊、鍵盤、檔案拖放等）檢查 `APP.state.readOnly`，若為 `true` 則阻擋操作並顯示提示訊息「🔒 伺服器中斷，唯讀模式不可操作」。 |
| **實作範圍** | 主要函式：`sendChatMessage`、`createNoticePost`、`editNoticePost`、`deleteNoticePost`、`togglePinNoticePost`、`addChecklistItem`、`toggleChecklistItem`、`deleteChecklistItem`、`deleteChecklistBoard`、`resetChecklistBoard`、`setChecklistReminder`、`handleFiles`、`sendNextFile`、`createKeyEntry`、`editKeyEntry`、`deleteKeyEntry`、`toggleKeyActive`、`setKeyProgram`、`useTemplate` 等。每個函式的開頭加入唯讀檢查，以及 UI 按鈕/輸入框的 click/keydown handler 中也加入檢查。 |
| **驗收條件** | - 在唯讀模式下點擊任何會修改資料的按鈕 → 無反應 + 顯示 `APP.showStatusMsg('🔒 伺服器中斷，唯讀模式不可操作')` <br> - 在聊天輸入框按 Enter → 同上 <br> - 拖放檔案至檔案區域 → 同上 <br> - 勾選 checklist 項目 → 同上 <br> - 管理員功能（admin）也需要一併攔截 |
