# Solution
我來設計並實作雙端持久化日誌系統，從計劃開始：

## 📋 設計原則
- 使用文件輪轉 (RotatingFileHandler)，24h 自動清理
- 統一日誌格式：`[ISO時間戳] [級別] [模組] 訊息`
- client → 寫本地 localStorage buffer + 可匯出 txt
- server → 寫 `logs/` 目錄，按日期分割
- 所有 `writeLog()` 調用自動同時輸出到文件

讓我先檢查現有的日誌基礎設施：我看到了完整的程式碼。現在我來設計一個完整的持久化日誌系統。

## 🎯 實作計劃

### 1. **Server-side** (`server.py`) — Python logging 模組 + 24h 輪轉
- 新增 `LOGS_DIR = logs/` 目錄
- 使用 `logging.handlers.TimedRotatingFileHandler`
- 格式：`%(asctime)s [%(levelname)-8s] %(name)s: %(message)s`
- 每天午夜輪轉，保留 1 份備份 (= 24h)
- 攔截所有現有 `print()` → `logger.info()`
- 啟動時自動清理超過 24h 的舊 log 檔

### 2. **Client-side** (`templates/index.html`) — localStorage buffer + export
- 修改 `writeLog()` → 同時 append 到 `vcc_client_logs` localStorage key
- 時間戳格式統一為 ISO 8601
- 限制 buffer 大小（1000 行），超出自動 trim 舊 500 行
- 新增「📋 匯出日誌」按鈕，生成 `.log` 檔案下載
- 可選：自動每 60 秒 flush 到 localStorage

### 3. **API 端點** (server.py)
- `POST /api/log` — client 可上傳日誌條目
- `GET /api/log/recent?lines=100` — 讀取最近的 server 日誌

### 4. **清理策略**
- Server: TimedRotatingFileHandler + `backupCount=1` (只留當天 + 昨天)
- Client: localStorage buffer 最多 1000 行，自動 trim
- 舊備份檔會在下次輪轉時被覆蓋
