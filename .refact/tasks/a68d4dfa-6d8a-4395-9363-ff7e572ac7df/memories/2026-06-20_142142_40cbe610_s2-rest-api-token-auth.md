# S-2: REST API Token Auth - Complete

Completed card S-2: REST API Token Authentication for signal_server.py

Changes made:
1. Added `_api_send_json()` helper function (async) - sends JSON responses with CORS headers
2. Added `POST /api/admin/login` REST endpoint - accepts `{"password": "xxx"}` and returns `{"token": "...", "success": true}`
3. Added auth guard for POST/PUT/DELETE on room sub-resources (notice, checklist, keymgmt) - checks `Authorization: Bearer <token>` header using `_verify_session()`
4. GET endpoints (health, state, chats) remain open
5. OPTIONS preflight already existed, now also covers Authorization header

Key design decisions:
- Auth helper `_api_send_json()` can be used for 401 responses but the routing code uses the existing `response_data, status_code` pattern for consistency with the rest of _mini_http
- Admin login endpoint doesn't require auth (user isn't logged in yet)
- Token verification reuses existing `_verify_session()` and `_generate_session()`
